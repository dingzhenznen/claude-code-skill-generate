# PRD 文档生成器

## 概述

本技能帮助生成结构化、专业的产品需求文档（PRD）。提供通用产品模板和 Web 应用专用模板，涵盖产品概述、用户分析、功能需求、技术架构、项目规划等完整章节。

## 使用工作流

### 1. 确定 PRD 类型

根据用户需求选择合适的模板：

**通用产品 PRD** - 适用于：
- 各类产品的需求文档
- 移动应用、桌面软件
- 硬件产品、IoT 设备
- 需要完整产品规划的项目

**Web 应用专用 PRD** - 适用于：
- SaaS 平台、企业内部系统
- 电商平台、内容管理系统
- 社交平台、在线工具
- 需要详细前后端架构说明的 Web 项目

### 2. 收集信息

与用户确认以下关键信息：
- 产品名称和核心定位
- 目标用户群体
- 主要功能模块
- 技术栈偏好（Web 应用）
- 项目时间表和资源

### 3. 生成 PRD 文档

基于用户提供的信息和选定的模板：
1. 读取对应的模板文件（`assets/prd-template.md` 或 `assets/prd-web-app.md`）
2. 根据用户信息填充模板中的占位符
3. 保留用户未提供信息的占位符，供后续补充
4. 输出完整的 PRD 文档

### 4. 文档优化

根据项目特点调整：
- 删除不适用的章节
- 扩展关键功能描述
- 补充技术细节
- 添加项目特定内容

## 模板资源

本技能提供两个专业的 PRD 模板，存储在 `assets/` 目录：

### assets/prd-template.md
通用产品 PRD 模板，包含：
- **产品概述**: 名称、定位、背景、愿景
- **目标用户**: 用户画像、需求分析
- **市场分析**: 市场规模、竞品分析、竞争策略
- **功能需求**: 功能概览、详细说明（含优先级、用户故事、交互流程）
- **技术架构**: 技术栈、系统架构、数据模型、API 设计、性能与安全要求
- **非功能需求**: 性能指标、兼容性、可访问性
- **验收标准**: 功能、性能、安全、用户体验验收
- **项目规划**: 开发阶段、人力资源、风险管理
- **成功指标**: 业务指标、技术指标、用户满意度
- **附录**: 术语表、参考资料、变更记录

### assets/prd-web-app.md
Web 应用专用 PRD 模板，额外包含：
- **页面结构**: 站点地图、核心页面详情（URL、权限、交互）
- **前端技术**: 框架选型、响应式设计、浏览器兼容性
- **后端技术**: RESTful API 设计规范
- **用户认证**: 认证方式、会话管理、权限角色
- **数据安全**: 传输加密、存储安全
- **性能优化**: 前后端优化策略、性能指标
- **SEO 优化**: SSR、Meta 标签、Sitemap
- **监控与分析**: 前后端监控工具
- **部署方案**: CI/CD 流程、基础设施
- **Web 特定验收**: 跨浏览器、响应式、性能、安全测试

## 使用示例

**示例 1 - 通用产品**
```
用户: 帮我生成一个智能家居控制系统的 PRD
助手:
1. 读取 assets/prd-template.md
2. 根据智能家居场景填充：
   - 产品定位：统一管理家庭智能设备
   - 目标用户：追求生活品质的家庭用户
   - 核心功能：设备控制、场景联动、数据分析
   - 技术栈：移动端 + 云平台 + IoT 协议
3. 输出定制化的 PRD 文档
```

**示例 2 - Web 应用**
```
用户: 为在线项目管理平台创建 PRD
助手:
1. 识别为 Web 应用，读取 assets/prd-web-app.md
2. 填充 Web 特定内容：
   - 页面结构：仪表盘、项目列表、任务看板、团队协作
   - 前端：React + Ant Design
   - 后端：Node.js + PostgreSQL
   - 认证：JWT + SSO
3. 输出包含完整技术架构的 Web 应用 PRD
```

## 最佳实践

1. **信息收集要全面**: 尽可能在生成前收集完整信息，减少后期补充
2. **保留占位符**: 用户未提供的信息保留模板占位符，方便后续填写
3. **删除无关章节**: 对于简单项目，可删除不需要的章节（如市场分析）
4. **突出重点**: 根据项目特点扩展关键章节，如技术难点、核心功能
5. **版本管理**: 建议在附录中维护变更记录，跟踪 PRD 迭代历史
